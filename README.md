# Визуализатор алгоритмов на графах

Интерактивное веб-приложение для пошаговой визуализации классических алгоритмов
на графах: обхода в ширину (BFS), обхода в глубину (DFS), алгоритма Дейкстры,
а также алгоритмов Прима и Краскала для построения минимального остовного дерева.

Приложение реализовано на чистых HTML, CSS и JavaScript с использованием
ES-модулей, без сторонних фреймворков и зависимостей.

## Онлайн-демонстрация

Приложение доступно онлайн по ссылке:

**https://tartaletka03.github.io/graph-visualizer/**

Демонстрация развёрнута через GitHub Pages из репозитория проекта. Открыть можно
в любом современном браузере; установка не требуется.

## Возможности

- Интерактивный редактор графа: добавление, перемещение и удаление вершин и
  рёбер, ввод веса ребра через модальное окно.
- Пошаговое выполнение любого из пяти алгоритмов с подсветкой вершин и рёбер.
- Автоматическое воспроизведение с настраиваемой скоростью.
- Вывод порядка обхода (BFS, DFS), таблицы расстояний и предков (Дейкстра),
  списка рёбер и суммарного веса остовного дерева (Прим, Краскал).
- Предустановленный пример графа для быстрого старта.

## Архитектура

Приложение разделено на независимые модули по принципу единственной
ответственности:

1. **Модель графа** (`core/Graph.js`) - единственный источник данных о
   структуре графа. Хранит вершины и рёбра, поддерживает карту смежности
   для доступа к соседям вершины за время, пропорциональное её степени
   (а не общему числу рёбер).
2. **Структуры данных** (`structures/`) - `PriorityQueue` (бинарная
   min-куча для Дейкстры и Прима) и `DSU` (система непересекающихся
   множеств для Краскала).
3. **Алгоритмы** (`algorithms/`) - каждый алгоритм реализован как чистая
   функция, принимающая граф (и стартовую вершину) и возвращающая массив
   «шагов» - описаний состояний графа на каждом этапе. Это позволяет
   унифицировать визуализацию.
4. **Визуализация** (`visualization/Visualizer.js`) - отрисовка графа на
   SVG и применение шагов: назначение классов состояний вершинам и рёбрам.
5. **Контроллер интерфейса** (`ui/UIController.js`) - связывает модель,
   визуализатор и алгоритмы с действиями пользователя.

## Алгоритмы и сложность

| Алгоритм  | Структура данных         | Временная сложность |
|-----------|--------------------------|---------------------|
| BFS       | очередь                  | O(V + E)            |
| DFS       | стек                     | O(V + E)            |
| Дейкстра  | бинарная куча (priority queue) | O((V + E) log V) |
| Прим      | бинарная куча            | O((V + E) log V)    |
| Краскал   | DSU + сортировка рёбер   | O(E log E)          |

## Запуск приложения

Приложение не требует сборки и установки зависимостей.

### Вариант 0. Онлайн (GitHub Pages)

Откройте в браузере ссылку: **https://tartaletka03.github.io/graph-visualizer/**

Это самый быстрый способ - ничего скачивать и запускать не нужно.

### Вариант 1. Открытие файла

Откройте файл `index.html` в любом современном браузере (Google Chrome,
Mozilla Firefox, Microsoft Edge, Safari).

> Примечание: при открытии файла напрямую через `file://` некоторые браузеры
> ограничивают загрузку ES-модулей. Если модули не загружаются, используйте
> локальный сервер (вариант 2).

### Вариант 2. Локальный сервер

Из корня проекта выполните любую из команд:

```bash
# Python 3
python3 -m http.server 8000

# Node.js (http-server)
npx http-server -p 8000

# Bun
bunx serve -p 8000
```

Затем откройте в браузере адрес `http://localhost:8000`.

## Запуск тестов

Тесты проверяют корректность модели графа, структур данных и всех пяти
алгоритмов, включая граничные случаи (несвязный граф, изолированные
вершины, равные веса рёбер, граф из одной вершины).

### В Node.js

```bash
node tests/run-all.mjs
```

Ожидаемый результат: 57 тестов, все проходят.

### В браузере

Откройте файл `tests/index.html` (через локальный сервер, как описано выше).
Результаты отобразятся на странице.

## Использование

1. Нажмите **«Загрузить пример»** для быстрого старта или создайте граф
   щелчками по холсту (вершины) и по двум вершинам подряд (рёбра).
2. Выберите алгоритм в боковой панели.
3. При необходимости выберите стартовую вершину.
4. Нажмите **«Запустить»**, затем **«Шаг вперёд»** для пошагового выполнения
   или **«Авто»** для автоматического воспроизведения.

## Системные требования

- Любой современный браузер с поддержкой ES-модулей (Chrome 61+, Firefox 60+,
  Edge 79+, Safari 10.1+).
- Для запуска тестов в Node.js - Node.js 14+.
